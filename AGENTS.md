# AGENTS.md — このリポジトリで作業するエージェントへの標準指示

このプロジェクトは **Slidev 製のプレゼンテーション**（ダーク × cyan/purple ブランド、GitHub Pages 公開）。
デザインの単一の真実は [`DESIGN.md`](./DESIGN.md)。**作業の前に必ず読むこと。**

## 必須プロトコル

1. **デザイン変更の前に `DESIGN.md` を読む。** 配色・フォント・トーン（§2 ブランド）は**不変／変更禁止**。
   底上げするのは余白・タイポ・整列・装飾の**規律**だけ（作り直しではなく規律の上書き）。

2. **フィードバックは必ずルール化する（徹底）。** ユーザーからの指摘は“その場の修正”で終わらせず、
   **一般化して `DESIGN.md` に追記**する（`DESIGN.md` §0.1）。1つ直したら、再発防止の規律を1つ書く。
   - 普遍原則 → §1–§8 ／ 再利用・テンプレート → §0・§9・§11 ／ リポジトリ固有 → §10。
   - 「あえて残す」逸脱は §8.1 に**根拠つき**で記録（後続が“違反”として戻さないため）。
   - コミットメッセージに**何を規律化したか**を1行残す。

3. **他プロジェクトへ移植可能に保つ。** `DESIGN.md` §0.2 の階層（A 普遍／B ブランド値/C 実装）を守り、
   再利用アセット（§11）と整合させる。新規プロジェクトは「`DESIGN.md` ＋ `style.css` ＋ §11 の資産を持ち込み、
   ブランド値だけ差し替え」で同テイストを再現できる状態を維持する。

4. **変更後は必ず検証する。** dev（`npm run dev`, http://localhost:3030）と、Pages 公開時は
   build（`slidev build --base /<repo>/`）の**両方**で、見切れ・重なり・折返し崩れ・余白不足・
   低コントラスト・ナビゲーション（矢印/クリック送り/用語ジャンプ）を確認（Playwright / 目視 QA を最低1周）。

5. **配布 PDF は検証の“締め”に必ず作り直す。** `handout.pdf`（クリック式タブの全状態を1ページずつ収録した配布物）は
   `slides.md`／`style.css`／`components/` の変更で**古くなる**。項4の検証を終えたら、配布・共有の前に
   **`npm run export:handout` を実行して再生成する**のを最後の手順にする（フルビルド＋ブラウザ巡回で数十秒＝
   保存ごとの自動同期はしない手動の締め工程。生成物は `.gitignore`＝コミットしない・§10）。

## 主要ファイル

- `slides.md` … デッキ本体（Markdown ＋ 最小 HTML）
- `style.css` … ブランドトークン＋全クラス（Slidev 自動読込）
- `components/` … `RevealTabs.vue`（クリック式リビール）/ `Glossary.vue` ほか
- `composables/glossary.ts` … 用語集データ＋共有 ref
- `global-bottom.vue` … 用語リンク（`.gterm`）の委譲ハンドラ
- `setup/main.ts` … GitHub Pages ベースパスのナビ修正（Pages 公開に必須）
- `DESIGN.md` … デザイン規律・再利用手順（**最優先で参照**）
